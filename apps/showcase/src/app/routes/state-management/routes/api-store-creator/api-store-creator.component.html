Api Store Creator

Easily create flux pattern stores to manage communication between an api and the application. Requests are made to the store which manages the crud operations and then updates the store accordingly.

There are 2 types of api stores. The first is an enitity store which is an array of object identical to what is returned from a database. The second is a non-entity store which is for simple objects and other primitives. Entity stores are the most common.

Install

Setup

1. Import into module

2. Set up api calls in a service

3. Set up store connections in the component

4. Wire up to template

Basics

The backbone of the store is the "state$" property which contains both the data in the store and the api state inside a single subscription. This allows you to perform complex UI operations such as showing loading indicators and errors without needing multiple subscriptions.

Sometimes you want to extract just the data out of the store without state. Each store type has some additional methods to do just that.
